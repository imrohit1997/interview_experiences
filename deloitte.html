Java 8 Coding (09:10:39)

2. Write Java 8 code to get the first non-repeating character from a string (e.g., "swiss").

Using Java 8 Streams, you can collect the character counts into a LinkedHashMap (to preserve insertion order) and then find the first character with a count of 1.

import java.util.LinkedHashMap;
import java.util.function.Function;
import java.util.stream.Collectors;

public class Main {
    public static void main(String[] args) {
        String input = "swiss";

        Character firstNonRepeating = input.chars()
            .mapToObj(c -> (char) c)
            .collect(Collectors.groupingBy(
                Function.identity(), 
                LinkedHashMap::new, 
                Collectors.counting()
            ))
            .entrySet().stream()
            .filter(entry -> entry.getValue() == 1L)
            .map(entry -> entry.getKey())
            .findFirst()
            .orElse(null);

        System.out.println("First non-repeating character: " + firstNonRepeating); // Output: w
    }
}

Java Sorting (09:16:28)

3. What is the difference between Comparable and Comparator?
  • Comparable: An interface used to define the natural ordering of objects (e.g., sorting employees by ID). It provides the compareTo(Object obj) method. A class must implement this interface to be sortable, meaning the sorting logic is bound to the class itself.
  • Comparator: An interface used to define custom or multiple sorting sequences (e.g., sorting employees by Name or Salary). It provides the compare(Object obj1, Object obj2) method. It is external to the class being sorted, allowing sorting based on various attributes without modifying the class source code.

Microservices generally communicate in two ways:

Handling Failures in Microservices (09:38:12)

6. What happens if you call another service, it calls a third, and there is a cascading failure? How do you handle this?

A failure in a downstream service can cause cascading failures where all threads in the upstream services become blocked waiting for a response.

Handling Strategy: Circuit Breaker Pattern (Resilience4j / Hystrix)

  • If a service fails continuously, the circuit breaker "opens" and stops sending requests to the failing service, returning a fallback response immediately instead of blocking threads.
  • It periodically allows a few requests through (Half-Open state) to check if the downstream service has recovered.
  • We also implement timeouts and bulkheads to limit the blast radius.

Java 8 Core Features (09:40:09)

7. What are the key features of Java 8?
  • Lambda Expressions: Anonymous functions that help write clean and concise code.
  • Functional Interfaces: Interfaces with exactly one abstract method (e.g., Runnable, Callable, Predicate).
  • Stream API: A powerful API for processing sequences of elements (e.g., map, filter, reduce) in a functional and declarative way.
  • Default and Static Methods in Interfaces: Allows adding new methods to interfaces without breaking existing implementations.
  • Optional Class: A container object used to avoid NullPointerException.
  • New Date/Time API: A comprehensive and thread-safe API (java.time package).
8. Why was a new Date and Time API added in Java 8?

The legacy java.util.Date and java.util.Calendar classes had several design flaws:

  • Not Thread-Safe: The old classes were mutable, making them unsafe to use in multi-threaded environments without synchronization. The new API (java.time.*) is completely immutable and thread-safe.
  • Poor Design: The old API was counter-intuitive (e.g., months were 0-indexed while days were 1-indexed). The new API follows domain-driven design with clear classes like LocalDate, LocalTime, and ZonedDateTime.
  • Formatting Issues: SimpleDateFormat was also not thread-safe. The new DateTimeFormatter resolves this.

Java 8 Features & Optional (09:41:11)

9. What methods from the Optional class are you using to check if a value is present or to return a non-value?

Optional is used to avoid NullPointerException.

  • Checking presence: isPresent() (returns true if value exists) and isEmpty() (Java 11+).
  • Returning a value or fallback:
    • orElse(T other): Returns the value if present, otherwise returns other.
    • orElseGet(Supplier<? extends T> other): Returns the value if present, otherwise invokes the Supplier to generate a value.
    • orElseThrow(): Throws an exception if the value is not present.

Java Functional Interfaces (09:42:25)

10. Explain Predicate, Supplier, and Consumer interfaces with examples.
  • Predicate<T>: Takes one argument and returns a boolean. Used for filtering. Example: Predicate<Integer> isEven = n -> n % 2 == 0;
  • Supplier<T>: Takes no arguments and returns a value. Used for deferred execution or generation. Example: Supplier<Double> randomValue = () -> Math.random();
  • Consumer<T>: Takes one argument and returns no result (void). Used to perform an action. Example: Consumer<String> printer = s -> System.out.println(s);

Executor Framework & CompletableFuture (09:44:44)

11. Are you using the Executor framework in your codebase? Can you tell me about CompletableFuture?
  • Executor Framework: Yes, used to manage thread pools (e.g., Executors.newFixedThreadPool(10)) to prevent the overhead of creating/destroying threads for every task.
  • CompletableFuture: Introduced in Java 8, it represents a future result of an asynchronous computation. Unlike standard Future, it can be explicitly completed and allows chaining of asynchronous tasks (e.g., thenApply, thenAccept, thenCombine) without blocking.

Exception Handling in Java Streams (09:46:54)

12. If an exception occurs in your Java streams code, how do you handle it?

Handling checked exceptions directly inside lambda expressions used in streams is tricky because lambdas like Function or Consumer don't declare thrown checked exceptions.

Solutions:

  1. Wrap the code in a standard try-catch block inside the lambda and handle it or throw a RuntimeException.
  2. Extract the logic into a separate method that handles the exception and call that method from the stream.
  3. Create a custom wrapper utility method (a "ThrowingFunction" interface) to encapsulate the try-catch block to keep the stream clean.

JUnit Testing & Private Methods (09:51:49)

15. If a public method calls multiple private sub-methods, how do you write JUnit tests to cover all those subsequent methods?

Best Practice: You should test the public method with various inputs and mock responses to hit all the conditional branches within the private methods. Private methods are implementation details and should ideally be tested through the public API.

If Direct Testing is Needed:

  • Use Reflection to make the private method accessible (Method.setAccessible(true)) and invoke it directly.
  • Use a library like PowerMock (though deprecated for newer Java versions) which allows testing/mocking private methods.
  • Alternatively, if the private methods are highly complex, it's a sign they should be extracted into a separate component/class where they become public and independently testable.

SOLID Principles (09:58:37)

17. Are you aware of the SOLID principles?

Yes, SOLID principles help in making the codebase scalable and maintainable:

  • Single Responsibility Principle: A class should have only one reason to change.
  • Open/Closed Principle: Software entities should be open for extension but closed for modification.
  • Liskov Substitution Principle: Derived classes must be substitutable for their base classes.
  • Interface Segregation Principle: Many client-specific interfaces are better than one general-purpose interface.
  • Dependency Inversion Principle: Depend on abstractions, not on concretions.

Java Collections: Sets (10:02:16)

18. Do you know the difference between HashSet, TreeSet, and LinkedHashSet? What does TreeSet use internally?
  • HashSet: Uses a Hash Table internally. Does not maintain any order. Offers O(1) performance for basic operations.
  • LinkedHashSet: Uses a Hash Table with a Doubly-Linked List. Maintains the insertion order. O(1) performance but requires more memory.
  • TreeSet: Maintains elements in sorted (natural or custom) order. Offers O(log N) performance for basic operations.

TreeSet Internals: Internally, a TreeSet is backed by a TreeMap, which in turn uses a Red-Black Tree (a self-balancing binary search tree).

Spring Boot Annotations (10:05:01)

19. What is the purpose of @SpringBootApplication? How do you create a custom annotation?

@SpringBootApplication: It is a convenience annotation that combines three annotations:

  • @Configuration: Tags the class as a source of bean definitions for the application context.
  • @EnableAutoConfiguration: Tells Spring Boot to start adding beans based on classpath settings, other beans, and various property settings.
  • @ComponentScan: Tells Spring to look for other components, configurations, and services in the current package and its sub-packages.

Custom Annotation: Created using the @interface keyword. We also define its scope using meta-annotations like @Target (e.g., METHOD, TYPE) and @Retention (e.g., RUNTIME to make it available via reflection).

Spring Stereotype Annotations (10:08:42)

20. What is the purpose of stereotype annotations (@Component, @Service, @Repository) in Spring Boot?

They are used to declare beans so Spring's component scanning can automatically detect them and register them in the Application Context.

  • @Component: A generic stereotype for any Spring-managed component.
  • @Service: A specialization of @Component indicating that the class holds business logic.
  • @Repository: A specialization of @Component for DAO (Data Access Object) classes. It additionally provides automatic translation of database exceptions into Spring's DataAccessException hierarchy.

Multiple Databases in Spring Boot (10:11:39)

21. How do you configure two databases in a Spring Boot project?

To configure multiple databases:

  1. Define separate data source properties in application.properties (e.g., spring.datasource.db1.* and spring.datasource.db2.*).
  2. Create separate @Configuration classes for each database.
  3. In each config class, manually create the DataSource, EntityManagerFactory, and TransactionManager beans.
  4. Annotate one configuration's beans with @Primary so Spring knows the default to use if not explicitly specified.
  5. Use @EnableJpaRepositories in each configuration class to point to specific packages containing the respective repositories.

Maven Dependencies & Conflict Resolution (10:13:17)

22. If you have two JARs with identical class paths or want to exclude classes from a huge dependency to reduce compilation time, how do you handle this?

In Maven, this is handled using the <exclusions> tag.

If Dependency A brings in an unwanted version of Dependency B (or an unnecessary huge JAR), you can exclude Dependency B from Dependency A's declaration in your pom.xml.

<dependency>
    <groupId>com.example</groupId>
    <artifactId>dependency-a</artifactId>
    <version>1.0</version>
    <exclusions>
        <exclusion>
            <groupId>com.example.unwanted</groupId>
            <artifactId>unwanted-jar</artifactId>
        </exclusion>
    </exclusions>
</dependency>

This prevents the unwanted classes from entering the classpath, avoiding class conflicts (like NoSuchMethodError) and reducing the build footprint.

23. If we include two custom JARs with the same class name, how will Maven and Java handle that?

This situation leads to a phenomenon known as "JAR Hell" or Classpath shadowing.

  • Maven's Role: Maven itself only resolves dependencies and downloads the JARs. If both JARs are explicitly declared or brought in transitively (and not excluded), Maven will put both JARs on the classpath.
  • Java's Role (ClassLoader): During runtime, when the JVM needs to load the class, the ClassLoader searches the classpath sequentially from left to right (or top to bottom). It will load the first class it encounters that matches the fully qualified name and completely ignore the subsequent identical class in the second JAR.

If the first loaded class is incompatible with what parts of your application expect (e.g., missing a method), it will result in runtime errors like NoSuchMethodError or NoClassDefFoundError. To fix this, you must use Maven's <exclusions> to ensure only the correct JAR is included on the classpath.

CI/CD & Deployment (10:16:21)

24. How does the CI/CD pipeline work? How gets it triggered in GitLab and what is the deployment strategy?

Triggering: In GitLab, the CI/CD pipeline is defined in a .gitlab-ci.yml file. When code is pushed to a repository, GitLab's runners automatically detect this file and trigger the pipeline stages (Build, Test, SonarQube scan, Dockerize). This is driven by webhooks internally.

Deployment Strategy: Common modern strategies include:

  • Rolling Update: Instances are replaced one by one with the new version, ensuring zero downtime (default in Kubernetes).
  • Blue-Green Deployment: Two identical environments (Blue and Green). Traffic is routed to the new version (Green) only after it's fully tested. Quick rollback is possible by just switching the router back.
  • Canary Deployment: The new version is deployed to a small subset of users first. If it is stable, it's rolled out to everyone gradually.
